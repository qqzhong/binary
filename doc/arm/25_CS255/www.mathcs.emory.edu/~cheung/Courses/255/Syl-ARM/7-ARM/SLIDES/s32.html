   Offset(value2) = 4
   Offset(value3) = 6 (4+2)
   Offset(next)   = 8 (4+2+2)

Final comment: offset computation is actually more complicated

  • The complete process to compute the offset of a field is more complicated due to memory alignment requirments:

      • An short type field must be placed at an address that is divisible by 2

      • An int type field must be placed at an address that is divisible by 4

      • Etc, etc

  • The memory alignment requirement will introduce gaps (= unused memory bytes) within the object

    The gaps makes the offset computation more complicated

  • In CS255, we will only use object definitions that has no gaps between fields.

How to obtain the base address of a List object

The linked list is chained by storing the base address of the next object in the next variable:

The variable head contains the base address for the first list element
The variable head.next contains the base address for the 2nd list element
The variable head.next.next contains the base address for the 3rd list element
And so on...
