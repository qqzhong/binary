The principle of accessing data stored in data structure

Principle of accessing data stored in the data structure:

  • The array data structure is designed for speed (sacrificing flexibility):

      • Every array element in an array can be access immediately     
      • Increasing the size of an array is difficult or even impossible

  • The linked list data structure is designed for flexibility (sacrificing speed):

      • Inserting (= increasing the size) into a linked list is easy
      • We must traverse (= take time !) a list to access the list elements

  • Accessing data in a data structure depends on how the data structure is organized

      • How to access a data structure depends on the structure/organization of the data structure

The principle of accessing a primitive variable inside a data structure
 

  • All data structures (array and linked list) contain at the most basic level:

      • Primitive typed variables         

  • To access a primitive typed variable (that is stored in the memory), you always need to know (= use):

      1. The memory location (address) of the (primitive typed) variable

      2. The data type (= number of bytes) used to store the (primitive typed) variable

        (You need to know which load/store instruction (ldrsb/ldrsh/ldr and strsb/strsh/str) to use)

Arrays in a high level programming language
 

  • High level programming languages provides the following expression to access one element in an array: 

          ArrayName[ index ]

    Examples: 

          int A[10];             

    A[3]    // index = 3
    A[i]    // index = i
    A[i*j]  // index = i*j

I will use A[i] to illustrate the array access technique

Information needed to access an array element A[i]
 

Information needed in order to access the array element A[i]:

  1. The memory location (address) of the array element A[i]

  2. The data type (= number of bytes) of the array element A[i]

How to find these information:

  1. The memory location (address) of the array element A[i] is calculated

  2. The data type (= number of bytes) of the array element A[i] is given by the array definition

How to find (= calculate) the address of the array element A[i]

How is an array stored in RAM:

 

 

We must find (= calculate) the address (= start location) of the variable A[i]

How to find (= calculate) the address of the array element A[i]

How to find (= calculate) the address of the array element A[i]:

 

 

Base address = the start of the array A[ ]
Offset = # bytes between base address and address of A[i]

How to find the base address of an array
 

Easy: use a label to mark the array location 

   A:  .skip N  (N depends on array type and size)

 

The label A will be equal to the base address of the array A

How to find the offset of A[i] from the base address
 

There are i array elements between the start of the array and the array element A[i]:

 
 

Each array element occupies the same number of bytes !!!

How to find the offset of A[i] from the base address
 

There are i array elements between the start of the array and the array element A[i]:

 
 

Offset of A[i] = i × sizeof(type of array A)
