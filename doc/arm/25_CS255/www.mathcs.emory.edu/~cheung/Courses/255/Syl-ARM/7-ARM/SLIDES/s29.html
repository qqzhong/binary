List (objects)
 

  • We will now study the storage and access of the (linked) list data structure

  • A list is an object in Java

    We will therefore how Java store objects first

How the compiler process a class definition
 

Example of a class definition: 

   public class myClass {
      int x;  
      int y;  
      int z;  

      // Methods omitted for brevity  


 
   }

The Java compiler will record (1) data type and (2) variable name of each instance variable in the class

How the compiler uses the information of a class definition
 

Example: 

   public class myClass {
      int x;  
      int y;  
      int z;  

      // Methods omitted for brevity  
   } 

   public class myProg {
      public static void main( ) {

          myClass x = new myClass( ); 
      }     
   }

When user creates an object (with the new operator), the Java run-time system will reserve contiguous memory cells to store all the instance variables in the object

How the compiler uses the information of a class definition 

   public class myClass {
      int x;  
      int y;  
      int z;  
   }
 

All objects of the same class are stored in the same structure (with same offsets !!!)

Access expressions used to access the components inside an object

Java's syntax to access (= use) the components inside an object: 

 objVar.x = the x component (int variable) in object objVar     
 objVar.y = the y component (int variable) in object objVar 
 objVar.z = the z component (int variable) in object objVar
 

All objects of the same class are stored in the same structure (with same offsets !!!)

How to access a component inside an object

Access technique to a component (= a primitive variable) inside an object:

Example:
         Offset(x) = 0        Note:   we still need the base address of the object !!
         Offset(y) = 4        Base addresses of list objects are maintained differently than arrays!!
         Offset(z) = 8
