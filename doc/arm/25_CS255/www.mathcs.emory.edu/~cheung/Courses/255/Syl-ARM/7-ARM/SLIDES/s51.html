Hansel and Gretel and non-leaf functions

In the story of Hansel and Gretel, they lost the bread crumbs that point the way back home because birds ate them:

This analogy is also applicable in function calls because:

  • You can lose your return address if you're not careful !!!        

How can you lost a return address - Example

Let's write this sequence of function calls using what we just learned: 

   public static void main(String[] args)
   {
      A( );
   }

   public static void A( )
   {
      ...
      B( );   // Calls B( )
      ...
   }

   public static void B( )
   {
      ...     // Does not call any method/function     
      ...
   }

How can you lost a return address - Example

The function call code: 

main:
        mov     r0, #1111
        mov     r1, #1111
        bl      A
        mov     r2, #2222
        mov     r3, #2222

Stop:
	nop

A:
        mov     r0, #3333
        mov     r1, #3333
        bl      B          // Overwrites lr (A's return address to main !)
        mov     r2, #4444
        mov     r3, #4444

        mov     pc, lr     // A fails to return to main !!!


B:
        mov     r0, #9999
        mov     r1, #9999

        mov     pc, lr     // B succeeds to return to A

How can you lost a return address - Example

Why the function A( ) cannot return to main( ): 

main:
        mov     r0, #1111
        mov     r1, #1111
        bl      A          // Saves return address to main in lr 
        mov     r2, #2222
        mov     r3, #2222

Stop:
	nop

A:
        mov     r0, #3333
        mov     r1, #3333
        bl      B          // A has lost its return address to main !
        mov     r2, #4444
        mov     r3, #4444

        mov     pc, lr     // Therefore: A can never return to main !!!


B:
        mov     r0, #9999
        mov     r1, #9999

        mov     pc, lr

Lesson learned from a non-leaf function
 

  • In non-leaf functions, we cannot leave the return addresses in the LR register unsaved

      • Because another bl instruction will overwrite the return address in the LR register

  • We must save the return address in non-leaf subroutines !!!

Fact:   there may be many levels of function calls

E.g.:   A( ) calls B( ), B( ) calls C( ), C( ) calls D( ), etc...

Therefore:   there are many return addresses that need to be saved (= stored)

Question:   how should we organized the saved return addresses ???

The order in which return addresses are used

Look at the order in which the return addresses are saved and used in a series of function call:

 
  • The last return address saved (= inserted) will always be the first return address used (= deleted)    (= LIFO order !)

The most efficient data structure to store return addresses
 

  • We must always use the most efficient data structure in computer programs

    (Because that will result in the most efficient programs)

  • The most efficient data structure is the one that closely corresponds to the insert/delete behavior (ordering) of the stored data !

      • The insert/delete ordering of return adresses is LIFO       

  • Therefore:

      • The most efficient data structure to store return addresses is:     

                  a   stack !!
