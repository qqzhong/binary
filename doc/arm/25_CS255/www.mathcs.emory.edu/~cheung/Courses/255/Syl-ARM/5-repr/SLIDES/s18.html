Review: the whole number data types byte, short and int

Previously, we have learned:

  • Integer (= whole number) data types used in a computer program are: 

       byte:  uses the  8 bits 2s complement code   
 short: uses the 16 bits 2s complement code
 int :  uses the 32 bits 2s complement code

 

The 2s complement code was developed to represent signed integer values so arithmetic operations can be implemented efficiently

Consequence (a very important fact):

  • To perform an arithmetic operations on an integer number, the integer number must be encoded in 2s complement !!

How does a program get binary numbers when I write decimal numbers in my code ?
 

Consider this program in a high level programming language: 

   int x;
   int main()
   {
      x = 15;   // 15 = 00001111 (bin) = 0F (hex)     
   }

Question:   how does the 15 become a (binary) 2s complement code (00001111) ???

  • Answer: the compiler translates every number you write into binary !!!

How does a program get binary numbers when I write decimal numbers in my code ?

Source code: 

   int x;
   int main()
   {
      x = 15;   // 15 = 00001111 (bin) = 0F (hex)     
   }

Hex dump of the object code:   (the 2s compl code for 35 is highlighted) 

0000000000000000 <main>:
   0:	55                   	push   %rbp
   1:	48 89 e5             	mov    %rsp,%rbp
   4:	c7 05 00 00 00 00 0f 	movl   $0x0f,0x0(%rip)
   b:	00 00 00 
   e:	b8 00 00 00 00       	mov    $0x0,%eax
  13:	5d                   	pop    %rbp
  14:	c3                   	retq   

  00 00 00 0f (Hex) = 00000000000000000000000000001111 (bin) = 15 (dec)

How does a program get binary numbers when I write decimal numbers in my code ?

Try using negative numbers: 

   int x;
   int main()
   {
      x = -15;   // What's the 2s compl for -15 ?     
   }

Hex dump of the object code:   (the 2s compl code for 35 is highlighted) 

0000000000000000 <main>:
   0:	55                   	push   %rbp
   1:	48 89 e5             	mov    %rsp,%rbp
   4:	c7 05 00 00 00 00 f1 	movl   $0xfffffffff1,0x0(%rip)
   b:	ff ff ff 
   e:	b8 00 00 00 00       	mov    $0x0,%eax
  13:	5d                   	pop    %rbp
  14:	c3                   	retq   

  ff ff ff f1 (Hex) = 11111111111111111111111111110001 (bin) = -15 (dec)

The computer is an expert in converting to 2s complement code !!
