5. The Skeleton Project File "pj4.s"

    The assignment consists of 10 easy questions given inside the assembler program pj4,s that was copied into your project directory through the preparation step above.

    You must edit the project file "pj4.s" and write the required assembler instructions in this program file.

    How to edit the project file pj4.s file:

      • Read this write up: click here                           

    The skeletal program has 10 sections, labeled as Q1, Q2, ..., Q10 where you put assembler instructions

  • Advice:

      • Do the project piece meal:

          • Write the assembler instructions for question 1 at label Q1
          • Then compile and run the program pj4.s

          • When Q1 is done, do Q2, and so on

      • When you test the assembler instructions for a question, use the STEP button (for explanation of the menu buttons, see this webpage: click here )

      • When you want to skip over an already tested section of the project, use the RUN button --- it will skip to the code of next question (for explanation of breakpoint, see this webpage: click here )

  • Assignment:

      • These instructions are also given inside pj4.s

      • Write the assembler instructions the each answer in the corresponding section of Q1, Q2, ..., Q10 in the pj4.s file. 

               Q1:    ans_i = 74;
   Q2:    ans_s = 39;
   Q3:    ans_b = -44;
   Q4:    ans_i = x;
   Q5:    ans_s = x;
   Q6:    ans_b = x;
   Q7:    ans_b = y;    (Can you explain why ans_b != y ?)
   Q8:    ans_s = y;    (Can you explain why ans_s == y ?)
   Q9:    ans_s = z;    (Can you explain why ans_s != z ?)
   Q10:   ans_i = y; 
  
Pre-defined variables used by the program:

   ans_b:  byte
   ans_s:  short
   ans_i:  int

   x:  -13    (byte  variable) binary: 11110011                            
   y:  2057   (short variable) binary: 00001000 00001001                  
   z:  147463 (int   variable) binary: 00000000 00000010 01000000 00000111

    Recall from the lecture(s) that:

      • Byte type variables are one byte in length,
      • Short type variables are two bytes, and
      • Integer type variables are 4 bytes long.

    Warning:

      • Be very careful and use the correct operand size in each assembler instruction.

        In other words: be careful when you load a value, make sure you used the correct ldr instruction:

          • Use ldrsb when you load a byte value
          • Use ldrsh when you load a short value
          • Use ldr when you load a int value

        And be careful when you store a value, make sure you used the correct str instruction:

          • Use strb when you store a byte value
          • Use strh when you store a short value
          • Use str when you store a int value

    There are some more helpful hints on testing your program in the next section !!!

6. Running/testing the program

    The main purpose of this assignment is to get you familiarized with the m68000 system. So follow the following instruction carefully.

    This is how you run your program assuming that you called your program file "pj4.s":

    1. Edit your assignment program (pj4.s) according to the instructions above...

        • You can complete 1 question at a time and compile and test the program.

    2. After you're have finished writting some part of the assembler program, you can run the program as follows:

        • First, assembler the program using EGTAPI.

          Read this webpage on how to assembler a single file project: 

              http://www.mathcs.emory.edu/~cheung/Courses/255/Projects/Egtapi/doc2019/compile.html

          You need to select pj4.s and click Compile.

          When the compilation is completed successfully, it will produce a listing in file pj4.list and an executable file in pj4.arm.

          The listing file is not used in the execution, but it does contain interesting information about the placement of the variables in the memory (symbol table).

        • If compilation is successful, you can execute the pj4.arm in EGTAPI

          Read this webpage on how to run a compiled program in EGTAPI: 

              http://www.mathcs.emory.edu/~cheung/Courses/255/Projects/Egtapi/doc2019/load.html

          Basically, you select pj4.arm and click Select.

        • If all is well, the program will stop at the breakpoint main.

          You will see this in the Breakpoint window:

          The label main is highlighted - indicating the program has stopped at the label main

          You will see the first instruction in your program in the Curr. Instr window.

          Now you are ready to test your program.

    3. To test your program, press the Step key (right arrow with a bar).

      Effect:

        • The Step will cause EGTAPI to execute the current assembler instruction and stop.

      Press Step a few more times to execute all the instructions that you have written to perform Question 1 of the assignment.

      If your program is correct, EGTAPI will:

        • show ans_i = 74 in the "Variable Watch" window at the top in the right most column of the EGTAPI window

    4. How to check for errors:

        • Press the Step button repeatedly (but SLOWLY !) to check the execution of your program.

        • While you are pressing the Step button, look at how the variables ans_b, ans_s and ans_i variables - specifically, look for the correct values moved in these variables in the "Variable Watch" window.

          Make sure the values are right (according to the questions in the project assignment).

    5. Program end:

        • EGTAPI will stop when it reaches the instruction at the Stop label.

          When EGTAPI reaches the Stop label, the Step (and other buttons) will no longer work.

          If you want to re-run the program, you can click on Reload.

          If your program is complete and correct, you can turn in the project.

    6. If you found an error (i.e.: you detects that the value stored in some variable is incorrect), then you must correct your error:

        • Edit your pj4.s file (Use EGTAPI, File Browser, select pj4.s and click Edit)

        • Compile the program after you finish editing (and saving the file)

          (Click File Browser, select pj4.s and click Compile)

        • Re-run the program.

          (Click Reload)

      If you discover more errors, just repeat the above steps until your program is error-free.
