What is "returning from a subroutine" ?
 

Returning from a subroutine: 

   function f1( )               function f2( )  // Subroutine 
   {                   +------->{         
       ...             |            ...  |
       ...             |            ...  |
       ...       call  |            ...  |
       f2( ) ----------+            ...  |
       ...   <--------------+       ...  |
       ...                  |       ...  V 
       ...                  +------ return;
   }                             }

Also called:   return to your caller

Means: go back to the statement/instruction immediately after the subroutine call

Returning from a leaf subroutine

Returning from a leaf subroutine:

  • A leaf subroutine will make a subroutine call

    I.e.: a leaf subroutine will not use a bl instruction !

  • Consequently:

      • The lr (link) register will contain the correct return address (because this register will not be updated by a leaf subroutine)

  • Therefore, we can use: 

         mov   pc, lr  // Update PC with return address

    to return to the caller !!

Returning from a leaf subroutine - Example
 

How function calls are implemented in assembler: 

   main:                     myMethod:
         ...                       ...
         ...                       ...         
         bl  myMethod              ...
         ...                       ...
         ...                       mov  pc, lr
         ...
         bl  myMethod
         ...            
         ...

Some observation on function definitions in HL and assembler prog langs

No function header in assembler code - only has a label (= function name) 

 High level programming language:

   public int  myMethod ( int p1, int p2 ) // Function header 
   {
      ... (statements)
   }


 Assembler programming language:

   myMethod:  // Label (function name)
          ... 
          ... (assembler instructions)
          ...

Some observation on function definitions in HL and assembler prog langs

No identifiable curly brace {...} in assembler code that defines the function body 

 High level programming language:

   public int  myMethod ( int p1, int p2 ) 
   {  
      ... // function body
   }  

 Assembler programming language:

    myMethod:  
          ...
          ... 
       mov  pc, lr  // End of the function body

The end of a function is very hard to detect in assembler code

(Assembler programmers typically write a lot of comments to explain their code to themselves)
