I will use this List object definition in my example 2 on how to access list elements using ARM assembler: 

   public class List
   {
      int   value1;   // Data field 1
      short value2;   // Data field 2
      short value3;   // Data field 3
      List  next;     // Link field  

      // Other stuff omitted for brevity
   }

 

In order to show you how to access data field in a linked list, I will need to create a linked list...

In a real computer program, List objects are created by the new operator

For brevity sake, I will create a linked list using assembler directives inside the .data section

Creating a linked list using assembler directives

We can create the fields in a list object using .byte, .2byte, .4byte assembler directives

Example of a 5 elements linked list: 

p0: .4byte  333         // Object [333,23,33,p2]
    .2byte  23, 33
    .4byte  p2
    .skip 4             // gap
p1: .4byte  111         // Object [111,21,31,p3]
    .2byte  21, 31
    .4byte  p3
2: .4byte  444         // Object [444,24,34,p4]
    .2byte  24, 34
    .4byte  p4
    .skip 4             // gap
p3: .4byte  222         // Object [222,22,32,p0]
    .2byte  22, 32
    .4byte  p0
    .skip 4             // gap
p4: .4byte  555         // Object [555,25,35,null]
    .2byte  25, 35
    .4byte  0           // 0 = null

The first list element is p1 !! (Can you see it ?)

Review:   how to calculate the offset of fields in a (list) object - Example 2

Given the following List (object) definition: 

   public class List {
      int   value1;   // Data field 1
      short value2;   // Data field 2
      short value3;   // Data field 3
      List  next;     // Link field  
   }

The offset of the fields in objects defined by the above class definition are: 

   Offset(value1) = 0
   Offset(value2) = 4
   Offset(value3) = 6 (4+2)
   Offset(next)   = 8 (4+2+2)

Linked list access example 1:   load data fields in the first object in the list into registers R1, R2, R3  

The variables value1, 2 and 3 are fetched using load instructions with their correct offsets: 

 ARM assembler instructions:       

   // Fetch: head.value1, 2 and 3

   // Preparation:
   //   r0 = base address


   ldr   r1, [r0, #0]//Offset(v1)=0
   ldrsh r2, [r0, #4]//Offset(v2)=4
   ldrsh r3, [r0, #6]//Offset(v3)=6

The preparation required is: r0 = base address of the first list object

Linked list access example 1:   load data fields in the first object in the list into registers R1, R2, R3  

The base address of the first list object is stored in the variable head: 

 ARM assembler instructions:       

   // Fetch: head.value1, 2 and 3

   // Preparation:
   //   r0 = head


   ldr   r1, [r0, #0]//Offset(v1)=0
   ldrsh r2, [r0, #4]//Offset(v2)=4
   ldrsh r3, [r0, #6]//Offset(v3)=6

Linked list access example 1:   load data fields in the first object in the list into registers R1, R2, R3  

Solution:   How to load (= fetch)   head.value1,2,3   into registers R1, R2, R3: 

 ARM assembler instructions:       

   // Fetch: head.value1, 2 and 3

   movw  r0, #:lower16:head
   movt  r0, #:upper16:head
   ldr   r0, [r0]    // R0 = head

   ldr   r1, [r0, #0]//Offset(v1)=0
   ldrsh r2, [r0, #4]//Offset(v2)=4
   ldrsh r3, [r0, #6]//Offset(v3)=6

We must start by fetching head in register R0:
