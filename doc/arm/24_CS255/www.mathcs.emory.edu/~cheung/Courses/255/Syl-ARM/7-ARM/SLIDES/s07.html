The (program) text section/segment of an assembler program

  • The .text assembler directive tells the ARM assembler to start assembling the line after the directive into the .text section of the computer program

  • The .text section of a computer program contains computer instructions:

How does a compiler use the .text section
 

  • A program consists of:

      • Variable definition        
      • Statements (inside functions)        

  • The statements in a high level language program corresponds to instructions in an assembler program

  • Therefore:

      • The statements in a high level language program are stored in the .text section

How will the instructions occupy the instruction section in memory

  • The compiler translates statements in a program into assembler codes

  • The assembler codes are translated by the assembler and placed at the beginning of the object code output file:

  • When the object code file is loaded into memory to be run, the instructions will occupy the instruction section in memory

Important (memory) locations in a computer program
 

  • Some instruction location (= address) are very important

    Examples:

      • The location of the first instruction of a method (e.g.: the main( ) method)
      • The start of a loop
      • etc

  • These important memory locations in the computer program must be identifiable and used by the assembler program !!!

We need a mechanism to mark the important memory locations in a computer program

Label: identifier for memory locations

  • The assembler uses a label to "mark" a memory location that has a special significance.

    Example: 

      main:                                      
     push    {lr}  // <--- marks this location
     push    {fp}    
     ....

  • A label can be added before any instruction in the assembler program

  • A label is a symbolic name for a memory location (= address)

    I.e.: a label is a symbolic name for a binary number !!!

How does the assembler associate a label with a memory address
 

  • The assembler (= translation program) will keep track of the memory locations that it uses to store the translated machine code in memory.

    I.e.:

      • The assembler knows the memory location where the next translated item will be put in the computer memory

  • When the assembler finds a label (such as main:) in the assembler program, the assembler will set the label equal to the memory address where it will store the next translated item
