Assembler instructions that manipulate the program stack
 

  • Every CPU provide assembler instructions to:

      • Push data stored in a register onto the program stack   

      • Pop data from the program stack into a register

Note:

  • The program stack is stored in the memory

  • We can (if we wanted to) push/pop the program stack using str/ldr instructions !

The push and pop instructions of ARM
 

  • The push and pop instructions of ARM has the following (simplified) syntax: 

          push  {rN}               

    pop   {rN}

    (The general syntax can use multiple registers, but I will only use 1 register in these instructions)

  • The push and pop instructions will always push or pop 4 bytes (on the stack or off the stack)

Achieving the same effect of the push instruction
 

We can achieve the same effect of the push {r0} instruction using these 2 instructions: 

main:
        mov     r0, #4  

	sub	sp, sp, #4  // Decrement stack pointer by 4 
        str     r0, [sp]    // Push value in r0 on stack      

        mov     r0, #9

	sub	sp, sp, #4  // Decrement stack pointer by 4 
        str     r0, [sp]    // Push value in r0 on stack
 

In fact, that is how the push instruction is executed internally inside the CPU

The pop {rN} instruction of ARM

Suppose the program stack contains the int value X at the top.
After excuting the pop {rN} instruction:

The register rN will contain the value X
The stack pointer (SP) will move down (= incremented) 4 bytes !

Example of the pop {rN} instruction

DEMO:   /home/cs255001/demo/asm/8-sub/push.s 

main:
        mov     r0, #4                       
        push    {r0}   // Push value in r0 on stack      

        mov     r0, #9
        push    {r0}   // Push value in r0 on stack

        pop     {r1}            // r1 = 9
        pop     {r2}            // r2 = 4

Achieving the same effect of the pop instruction
 

We can achieve the same effect of the pop {r1} and pop {r2} instructions using these 2 instructions: 

main:
        // pop {r1}
        ldr     r1, [sp]    // Pop top of stack into reg r1 
	add	sp, sp, #4  // Increment stack pointer by 4 
     

        // pop {r2}
        ldr     r2, [sp]    // Pop top of stack into reg r2 
	add	sp, sp, #4  // Increment stack pointer by 4
 

In fact, that is how the pop instruction is executed internally inside the CPU
