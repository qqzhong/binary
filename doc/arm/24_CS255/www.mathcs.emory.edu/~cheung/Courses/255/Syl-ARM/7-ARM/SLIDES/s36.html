Recall how an out-of-order program flow happens
 

Visit this class note to review how an out-of-order program flow happens:

  • Out-of-order program flow - branching

 

In a nutshell:   the current instruction updates the program counter (PC)

The effect of a branch instruction
 

The branch instruction will update the program counter: 

        .text
main:
        mov     r1, #1111    // Instructions executed in sequence
        mov     r2, #1111    // "normal" program flow
        mov     r3, #1111

        b       there        // Updates PC, set PC = address there

        mov     r4, #2222    // These instructions are SKIPPED over !
        mov     r5, #2222
        mov     r6, #2222

there:
        mov     r4, #4444    // Out of order program flow
        mov     r5, #4444
        mov     r6, #4444

Simulating the branch instruction with a series of assembler instructions
 

We can achieve the (same) effect of the branch instruction using 3 (known) instructions: 

        .text
main:
        mov     r1, #1111    // Instructions executed in sequence
        mov     r2, #1111    // "normal" program flow
        mov     r3, #1111

        movw  r0, #:lower16:there // Updates PC, set PC = address there
        movt  r0, #:upper16:there
        mov   pc, r0

        mov     r4, #2222    // These instructions are SKIPPED over !
        mov     r5, #2222
        mov     r6, #2222

there:
        mov     r4, #4444    // Out of order program flow
        mov     r5, #4444
        mov     r6, #4444
