Review: memory addresses are binary numbers !

Memory cells (bytes) are identified by a unique address:

A memory address in a ARM assembler program is always expressed as a 32 bits binary number (see the address bus) !!!

How do you move (= put) a memory address value into a register in ARM
 

  • A memory address must always be treated as large number !!!

  • Therefore:  

      • You must use 

             movw  rN, #:lower16:memAddress
   movt  rN, #:upper16:memAddress

        to store the memory address memAddress into a register

Review:   labels identifies (important) memory addresses in an assembler program

  • In assembler programs, we use labels to identify important memory locations (= addresses) 

         a:   ....  // a identifies this memory location

  • Labels are symbolic names for memory addresses (= binary numbers):

                     

How to move memory addresses into a register in ARM
 

  • Suppose the label a is the symbolic name for the memory address of the (any type of) variable a: 

         a:  .4byte 15

  • If you want to move (= put) the memory address a (it's a binary number !) into the register r0, you can use: 

         movw  r0, #:lower16:a 
   movt  r0, #:upper16:a
