Always use header files to store the struct's and typedef's in your C program

  • struct and typedef definitions

    • Because struct definitions and typedef clauses must preceed all variable definitions:

        • A very common practice in C programming is to put struct definitions and typedef clauses in (one of more) header files

        • And include the header files with struct definitions and typedef clauses as the first include files !!!

          (Because the type definitions must preceed all variable definitions and variable declarations)

    • Example:

        Header file: types.h Program file: use-types.c 
        typedef struct BankAccount   
{  
   int    accNum;
   double balance;
}  
BankAccount_t;













        #include <stdio.h>

#include "types.h" // Include my type definitions

BankAccount_t b;   // Now we can define global vars

int main(int argc, char *argv[])
{
   struct BankAccount a;  // And also define local variables

   a.accNum = 123;
   a.balance = 4000;

   printf("a.accNum = %d  a.balance = %f\n", a.accNum, a.balance);

   b = a;
   printf("b.accNum = %d  b.balance = %f\n", b.accNum, b.balance);
}
