High level language and low level language programming

  • Important fact:

      • The computer cannot (directly) execute a computer program written in a high level programming language !!

  • In CS170/CS171:

      • You must compile (= translate) your Java (high level language) program into an "object" (.class) file before you can run it

    A compiler (e.g.: javac) will translate the statements (commands) written in a high level programming language into a series of machine instructions that achieves the same result of the statements.

  • Important difference:

      • The computer can (directly) execute a program in computer (machine) instructions !!

Example to show you the entire program compilation process - DEMO

The programmer write a program in a High level language program (C language): 

 int x, y, z;

 int main( )
 {
   z = x + y;
 }

The compiler will translate the (C) program into an equivalent program written in assembler code: (Typically: 1 statement will general many instructions !!) 

main:
        str     fp, [sp, #-4] // These assembler codes
        add     fp, sp, #0    // perform: z = x + y
        ldr     r3, .L3
        ldr     r2, [r3]
	... (many more, omitted for brevity)

An assembler will then translate the assembler codes into (binary) machine instruction codes: 

00000000 <main>:
 0:   e52db004  <-- binary number written in hexadcimal 
 4:   e28db000  // These are (binary) machine codes !!!
 8:   e59f3028   
 ...
