Review: the (in)famous Hello World program (in C)
 

The Hello World program in C: 

  #include <stdio.h>

  int main( )
  {
     printf("Hello World\n");
  }

The Hello World program in ARM assembler ( pull out ) 
       .global main    // "Export" the identifier "main"

       .text
main:                  // Start of program
        push    {lr}   // Save the return address on the stack
        push    {fp}   // Save the frame pointer on the stack
                       // Explained later in CS255

        movw    r0, #:lower16:HelloStr
        movt    r0, #:upper16:HelloStr
        bl      printf  // Call printf function     

        pop     {fp}    // Pop the frame pointer
        pop     {pc}    // Pop the return address into PC
                        // Explained later in CS255

       .data
HelloStr:                      // Label marking this location
        .asciz "Hello World\n" // The Hello string
        .end

Every assembler program starts executing at the instruction at address marked by main:

Structure of a computer program

A program consists of instructions (statements) and data (constants and variables) 

       .global main

       .text    (starts the instructions section)

I will now explain the elements in the Hello World assembler program

Explanation of line 1: .global main

Besides the instructions and the variables , we can have the following constructs in an assembler program:

  • Assembler directives: (e.g.: .global)

      • Assembler directive = a command/request to the assembler (= translator) asking the assembler to "do something"

    We will study the essential assembler directives in this set of slides

  • Labels (e.g.: main)

      • A label = an identifier in an assembler program that is used to mark (identify) the memory location (= address) of:

          1. an instruction           or           
          2. a constant/variable         

The .global assembler directive
 

  • .global main:

      • The .global directive instructs the assembler (= translating program) to make the label main accessible to the linker application

  • main is a label used to mark the memory location (= address) of the first instruction of a (Java/C) program

  • Linker = an application that combines multiple program files into one executable program

      • The linker will need to know the memory locations of some important labels in order to combine different programs !

Background information: Program organization (arranged by the assembler)

For efficiency reason, the assembler will store similar things in a computer program together.

All program instructions are stored at the start of the memory called the instruction section:

Background information: Program organization (arranged by the assembler)

The constants used in the program and the permanent variables are stored after the instructions

This area of the memory is called the data section (or data segment):

Background information: Program organization (arranged by the assembler)

The rest of the memory are unused (= available) - some of these memory cells may be used later when the program is running

The .text assembler directive
 

  • .text:   instructs the assembler (= translating program) to assembler (= translate) the subsequent items (usually program instructions) in the instruction section (or instruction segment) of the computer program

  • Program lines that follows the .text keyword are stored in the instructions section

    Example: 

        .text     // Store what follows in the instructions section 

  push {lr} <-- Stored in the instructions section/segment   
  push {fp} <-- Stored in the instructions section/segment
  ...

The .data assembler directive
 

  • .data:   instructs the assembler (= translating program) to assemble constants and variable definitions

  • Program lines that follows the .data keyword are stored in the constants and permanent variables section

    Example: 

        .data

  .asciz "Hello World\n" // Stored in the 
                         // constants section

The .data keyword is another assembler directives (= commands given to the assembler program)

The .end keyword
 

  • .end:   terminates the assembler session

  • Program text that follows the .end keyword is ignored
