Working with ASCII code in Java

  • Characters in a programming language

    • Characters:

        • Characters (a.k.a. character constants) are denoted using single quotes: 

               'A'   'B'   'C'    '1'    '2'   ...

        • The type of a character is: 

                char

        • The value of a character is:

            • The value of the binary number ASCII code that corresponds to the character

    • Example:

        • 'A':

            • The data type of the character constant A is: char

            • the value of the character constant A is: 6510 (or 01000001(2) in binary)

          See the ASCII code table on this webpage: click here

    • Quiz 1:

        • Consider the following program: 

               char c;

   c = 'A';

          Question:

            • What is the bit pattern stored in the variable c ?      

        • Answer: 

                 01000001

    • Quiz 2:

        • Consider the following program: 

               byte c;

   c = 65;

          Question:

            • What is the bit pattern stored in the variable c ?      

        • Answer: same as quiz 1 !!! 

                 01000001

    • Note:

        • The difference between:

            • The character constant H that is represented by the binary pattern 01001000
            • The unsigned value 7210 that is represented by the same binary pattern 01001000

          is:

            • The data type !!!

              (Remember the Java compiler use the data type as context to interpret (= decode) the values !!!)

      Example that show you the same binary number is "interpreted" differently: 

              char c;
      byte b;

      c = 65;
      b = 65;

      System.out.println("char c = " + c);   // Prints "A"
      System.out.println("byte b = " + b);   // Prints "65"
      
      System.out.println();
      System.out.println("How to change 'interpretation' of a binary number");      

      System.out.println("char c casted by (byte) = " + (byte)c);
      System.out.println("byte b casted by (char) = " + (char)b);

      Output: 

        char c = A
byte b = 65

How to change 'interpretation' of binary number
char c casted by (byte) = 65
byte b casted by (char) = A

  --- You can see that variables c and b contains the SAME binary number !!!   
  --- By putting a casting operation, the number 65 is printed different
      (because a different println( ) function is called using different
      data type - through the overloading feature in Java

  • Casting (= conversion) between character typed value and integer typed value

    • Sometimes, we need to convert between char and another data type

    • In Java, we can cast between char and byte (or short or int) data types: 

            byte b;

    char c;

    b = (byte) c;     // Casts character value in c to byte type     

    c = (char) b;     // Casts byte value in b to char type

      Why do you want to do this:

        • We will need to String concentation operations on the binary ASCII code as a character

          In that case, we need to cast the binary number (used for computation) to char so Java will understand that we want to concatenate characters

          Example: we want to concatenate the letter 'A' to a string 

                 String s;

     s = s + 65;         <--- Concatenates "65" to a string 

     s = s + (char)65;   <---- Concatenates 'A' to a string     
                         // This is the same as: s = s + 'A' !!!
