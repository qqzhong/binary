  • Modulo arithmetic

    • Modulo arithmetic is an arithmetic technique where:

        • You first perform the arithmetic operation
        • Then reduce the result by subtracting a fixed value until the result is between 0 and that fixed value

    • Example: module 9 arithmetic 

            3 * 4 (modulo 9) = 12 (modulo 9) = 3       // 12 - 9 = 3
    7 * 4 (modulo 9) = 21 (modulo 9) = 3       // 21 - 9 - 9 = 3

      In modulo 9 arithmetic, the fixed value used is 9.

      After computing the result of some computation, we must reduce the result to a value between 0 .. 8 by subtracting 9 repeatedly until we get a value between 0 .. 8

  • How to read (code and decode) the (3 digit) 10s complement (odometer) code

    • In order to effectively use the odometer code, you need to be able to:

        • Encode any valid value using the odometer code
        • Decode an odometer code (= translate the odometer code to its corresponding decimal value)

      Let's learn to do these 2 things first....

  • Encoding a (valid) value using the odometer code

    • If you look at the positive values in this table: 

          Odometer reading:   500  ... 996 997 998 999 000 001 002 003 ...   499
  -----------------    +--------+---+---+---+---+---+---+---+---------+
  Value represented: -500  ... -4  -3  -2  -1   0   1   2   3  ...   499

      you can see that:

        • The odometer code for a positive value is obtained by adding preceeding 0's to the value

    • If you look at the negative values in this table: 

          Odometer reading:   500  ... 996 997 998 999 000 001 002 003 ...   499
  -----------------    +--------+---+---+---+---+---+---+---+---------+
  Value represented: -500  ... -4  -3  -2  -1   0   1   2   3  ...   499          

    Odometer reading              value represented by odometer reading
                 500 = 1000 - abs(-500)
	         996 = 1000 - abs(-4)
		 997 = 1000 - abs(-3)
		 998 = 1000 - abs(-2)
		 999 = 1000 - abs(-1)

      you can see that:

        • Odometer code = 1000 − abs(negative value)           

    • Algorithm to convert a value v to its 3 digit odometer (= 10s complement) code:

        • If value v ≥ 0:

            • just write the number out in 3 digits with leading 0

        • If value v < 0:

            • Compute 1000 − abs(v)
            • Write the result out in 3 digits with leading 0 digits       

      Example: 

         v = 3     3 digit 10's complement representation: 003
 v = 103   3 digit 10's complement representation: 103

 v = -3    3 digit 10's complement representation: 1000 - 3   = 997
 v = -103  3 digit 10's complement representation: 1000 - 103 = 897

  • Decoding an odometer code (= 10s complement code)

    • We first note that in the 3 digit 10s complement code: 

          Odometer reading:   500  ... 996 997 998 999 000 001 002 003 ...   499
  -----------------    +--------+---+---+---+---+---+---+---+---------+
  Value represented: -500  ... -4  -3  -2  -1   0   1   2   3  ...   499

      that:

        • Code words that begins with the digits 0, 1, 2, 3, 4 represent positive values
        • Code words that begins with the digits 5, 6, 7, 8, 9 represent negative values

    • The decoding algorithm is the reverse of the previous algorithm

      To convert a 3 digit 10's complement code c to its corresponding value

        • If the 10s complement code c begins with 0, 1, 2, 3 or 4 (i.e.: c represents a positive value):

            • The value represented by c is the "face value" (without the leading 0's)         

        • If the 10s complement code c begins with 5, 6, 7, 8 or 9 (i.e., c represents a negative value):

            • The value represented by c is equal:       − (1000 - c)           

      Example: 

         code c = 413  -> it is a positive number
		  the value = 413

 code c = 613  -> it is a negative number...              
		  the value =  - (1000 - 613) = -387

  • How is this "magic" achieve ?

    • Modulo arithmetic can be represented using a circle.

      The 3 (decimal) digit 10s complement code uses the modulo 1000 arithmetic.

      We can represent the modulo 1000 arithmetic on a circle as follows:

    • Additing is the same an moving a number of steps in the clock wise direction:

    • Subtraction is the same an moving a number of steps in the anti-clock wise direction:

      Notice that the value −1 is represented by the code 999.

      The reason is:

        • When we add the value −1, we must walk 1 position in the anti-clock wise direction

          Example: 2 + (−1) = 2 − 1 = 1

          Using the 10s complement code representation, we can obtain the correct representation for the result by walking 999 positions in the clock wise direction:

          Therefore:

            • If we represent the value −1 using the code 999, we can add without needing to use any gimmicks !!!

      So the 10s complements code uses the modulo arithmetic to obtain the correct representation for arithmetic operations !!!

      (Since multiply is repeated addition, it also works !!!)

  • Overflow

    • Fact:

        • Every Mathematical code has the overflow problem:

            • An overflow has occured is the result of a computation exceeds the range of the represented values in a code

    • Example: the 3 (decimal) digit 10s complement code can represent the value −500..499: 

          Odometer reading:   500  ... 996 997 998 999 000 001 002 003 ...   499
  -----------------    +--------+---+---+---+---+---+---+---+---------+
  Value represented: -500  ... -4  -3  -2  -1   0   1   2   3  ...   499

      Suppose we compute: 

           Add the values:    499 + 2         

   In 10s complement code:       
                             499     // repr the value 499
                           + 002     // repr the value 2
                           -----
			     501   --->  represents the value -499 !

    • What to do when there is overflow ?

        • You will need to use a longer (= more digit) code when overflow occurs.

      Longer codes means more data to store and you need more time to perform the computation

      That's why the computer provide different length codes to represent integers

      But instead of using 10s complement codes, the computer (because it uses binary numbers) uses 2s complement codes
