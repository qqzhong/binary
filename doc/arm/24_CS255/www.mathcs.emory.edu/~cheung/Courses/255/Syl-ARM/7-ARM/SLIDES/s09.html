Recall:   combining adjacent memory cells
 

Recall the restriction on combining 2 adjacent memory cells:

  • 2 adjacent memory bytes (8 bits each) can be combined into one memory cell of 16 bits if the address of the first byte is even (= divisible bt 2)

          

Recall:   combining adjacent memory cells
 

Recall the restriction on combining 4 adjacent memory cells:

  • 4 adjacent memory bytes (8 bits each) can be combined into one memory cell of 16 bits if the address of the first byte is divisible bt 4

          

Alignment requirement
 

  • Due to these "bytes combination" limitation, many computers have placement constraints on variable locations in memory

  • These placement constraints are called:

      • memory alignment requirements        

Memory alignment requirements in computer programs
 

  1. int typed and float typed variables must start at a memory address that is divisible by 4

  2. short typed variables must start at a memory address that is divisible by 2

  3. There are no alignment requirements for byte typed variables.

    Byte typed variables can be place anywhere in memory.

Memory alignment assembler directive

Assembler directive used to satisfy the memory alignment requirements: 

 .align  n  // If necessary, skip forward to 
            // an address that is divisible by 2n
 

How to use the .align directive: 

 // How to define an int type variable
 // with proper alignment requirement

    .align 2   // Skip forward to an address div by 22 = 4  
 i: .4byte 55  // Reserve 4 bytes for int typed variable

Note:   You will not need to use .align in the assembler programs (it's too low level)
