Converting a decimal sign-magnitude code into a (8 bits) 2s complement code
 

How to encode a positive value into a (8 bits) 2s complement code: 

  2s comp   Decimal                   Binary
  Code      Sign-Magnitude            Sign-Magnitude     
  ========================            ================
      ....                                  ...
  11111110    -2			    -10
  11111111    -1			     -1
  00000000     0			      0  
  00000001     1			      1  
  00000010     2			     10  
  00000011     3			     11  
  00000100     4			    100  
  .....                 34 ???              ...  
  01111111   127                        1111111

Converting positive values (and ZERO):
      (1) Convert the decimal sign-magnitude code into binary (by dividing by 2)
      (2) Prepend the resulting binary sign-magnitude code with leading ZEROs until you have the required number (= 8) bits

Example: Converting 34 into a (8 bits) 2s complement code

(1) Convert the decimal sign-magnitude code (34) into binary (by dividing by 2) 

            34
      2 --------- 0        34(10) = 100010(2)
            17
      2 --------- 1               (binary sign-magnitude repr)    
             8
      2 --------- 0
             4
      2 --------- 0
             2
      2 --------- 0
             1
      2 --------- 1
             0

(2) Prepend the resulting binary sign-magnitude code with leading ZEROs until you have 8 bits 

  binary sign-mag repr     2s compl repr (fixed length) 
 ----------------------   ---------------
        100010                00100010

Note: to convert to a 16 bits 2s complement repr, we must prepend more leading 0's (8 more) !!

How to negate a 2s complement representation

Problem statement:

  • Suppose x is the 2s complement code for some value v

    Find the 2s compl code "(-x)" that represents the signed value   –v

To find the solution, examine to mapping: 

  2s comp   Decimal
  Code      Sign-Magnitude            
  ========================
  10000000  -128  
  10000001  -127
	.....       
  11111100    -4
  11111101    -3
  11111110    -2
  11111111    -1      Notice: x + (-x) = 00000000           
  00000000     0      I.e.:   (-x) =  00000000 - x
  00000001     1      We will end up computing:
  00000010     2              (-x) = 100000000 - x
  00000011     3
  00000100     4
  .....            
  01111111   127

How to negate a 2s complement representation

Examine to mapping: 

  2s comp   Decimal
  Code      Sign-Magnitude            
  ========================
	.....       
  11111100    -4
  11111101    -3
  11111110    -2
  11111111    -1      Notice: x + (-x) = 100000000           
  00000000     0      I.e.:   (-x) = 10000000 - x
  00000001     1
  00000010     2
  00000011     3
  00000100     4
  .....

Solution: 

 8-bits 2s compl code for −x  =  100000000  −  2s compl code for x

Example: 

      100000000  
    -  00000011     (2s compl code for 3)
    -----------
       11111101     (2s compl code for -3)

Easier way to negate a 2s complement representation

Previously: 

 8-bits 2s compl code for −x  =  100000000  −  2s compl code for x

Easier way to compute the difference 100000000 − anyBinNumber: 

         100000000    −  x              Example flip-bits(x)
  <==>  (11111111+1)  −  x                         11111111
  <==>  (11111111     −  x ) + 1                 - 01011010
  <===>    flip-bits(x)      + 1                -----------
                                                   10100101

Example: 

   Original method            Easier method

         100000000                  flip-bits
       -  00000011            00000011 ---> 11111100
       -----------                                 1

Converting 2s compl code → decimal sign magnitude representation

Problem statement:

  • Given a 2s complement code (e.g.: 10101010

    What is the decimal (sign-magnitude) representation ?

    I.e.: what value does the 2s complement code represent ?

To find the solution, examine the mapping: 

  2s comp   Decimal
  Code      Sign-Magnitude            
  ========================
  10000000  -128  
  10000001  -127
	.....       
  11111100    -4     // Negative values are harder to see    
  11111101    -3
  11111110    -2
  11111111    -1               
  00000000     0      
  00000001     1
  00000010     2
  00000011     3
  00000100     4      // Positive values are "easy" to see
  .....               // Same as the bin number system !!
  01111111   127

We must split into 2 cases: (1) positive 2s compl codes and (2) negative 2s compl codes

How to tell if a 2s complement code represents a positive/negative value ?
 

Look at the left-most digit in the 2s complement code: 

  2s comp   Decimal
  Code      Sign-Magnitude            
  ========================
  10000000  -128  
  10000001  -127
	.....       
  11111100    -4     // Negative values           
  11111101    -3
  11111110    -2
  11111111    -1               
  00000000     0      
  00000001     1
  00000010     2
  00000011     3
  00000100     4      // Positive values 
  .....               
  01111111   127

Left-most bit = 1 in 2s complement code ⇒ represents a negatve value
Left-most bit = 0 in 2s complement code ⇒ represents a positive value (or ZERO)

Converting 2s compl code of a positive value → decimal sign magnitude representation
 

Algorithm:   just use the method to convert binary numbers → decimal number

Example:

  • Find the decimal (sign-magnitude) representation for the 2s complement code 00010010

  • Solution: 

        00010010  -> 0 at left most position ==> positive number 

  Apply binary number → deciaml number algorithm:

   000100102 = 0×7 + 0×6 + 0×5 + 1×4 + 0×2 + 0×2 + 1×1 + 0×0  
             = 16 + 2 = 18

Converting 2s compl code of a negative value → decimal sign magnitude representation

Algorithm:   (1) negate the 2s complement code
                      (2) convert the positive 2s compl code using the previous method
                      (3) add a negative sign to the asnwer

Example:

  • Convert the 2s compl code 11101110 to human readible decimal sign-magnitude: 

        11101110  -> 1 at left most position ==> negative number 

  (1) Negate:         100000000
                    -  11101110
                   ------------
                       00010010   <--- postive 2s compl code

  (2) (Pos) binary number -> decimal number:

       000100102 = 0×7 + 0×6 + 0×5 + 1×4 + 0×2 + 0×2 + 1×1 + 0×0  
                 = 16 + 2 = 18
  (3) Answer: −18
