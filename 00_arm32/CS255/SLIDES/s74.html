The fac( ) function revisited
 

Students are usually taught the following fac( ) function: 

   public static int fac(int n)
   {
      if ( n == 0 )
         return 1;
      else
         return (n * fac(n-1));
   }

This form does not reveal how the divide-and-conquer technique was used...

The fac( ) function revisited
 

I teach students the following function in CS170: 

   public static int fac(int n)
   {
      int helpSol;
      int mySol;

      if ( n == 0 )
         return 1;
      else
      {
         helpSol = fac(n-1); // Delegate a smaller problem to 
                             // someone else (not myself !) 
	 mySol = n*helpSol;  // Solve my problem using the 
                             // solution of the smaller problem 
         return (mySol);
      }
   }

This form reveals how the divide-and-conquer technique was used !

Important insight to help you understand and write recursive functions

You must distinguish between fac( ) that solves your problem and fac( ) that solves a smaller problem: 

   public static int fac(int n)  <--- This fac( ) is YOU
   {
      int helpSol;
      int mySol;

      if ( n == 0 )
         return 1;
      else
      {
         helpSol = fac(n-1);   <--- This fac( ) is someone else ! 
	 mySol = n*helpSol; 
         return (mySol);
      }
   }

So recursion will run another (copy of the) function (you can see the proof in the different stack frames created for each function call !)
