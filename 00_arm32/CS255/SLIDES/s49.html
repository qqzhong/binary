Assembler Instructions used to implement subroutine calling + return
 

  • All CPUs provides 2 assembler instructions to enable computer programs to call a subroutine and return to its caller:

      1. An "Call Subroutine" assembler instruction that is used by the caller to invoke a method which save the return address (somewhere)

      2. An "return to caller" assembler instruction that is used by the callee (= the called method) to return to its caller

We will now discuss the branch and link assembler instruction in ARM used to call a subroutine (= function)

The branch-and-link assembler instruction
 

Syntax and meaning (= effect) of the branch-and-link instruction: 

  bl  label       

  Effect:
        (1) Save the Program Counter (PC) into 
            the Link Register (LR)

	    Note: at the moment that the CPU executes 
                  the bl instruction, the PC
		  contains the address of the instruction
		  that follows the bl instruction !
	          

	(2) Branch to memory location marked by 
            the label "label"

A note on the value in the Program Counter (PC)
 

  • Recall that the Program Counter (PC) is increased after the CPU has fetched the next instruction -- see: click here

    So when the CPU executes the current instruction, the program pointer (PC) contains the address of the "next" instruction (that follows the current instruction

  • Therefore, when the CPU executes a bl label, the PC is pointing here: 

        bl label  // When the CPU executes the bl instr,
  instr     // <--- PC contains this address !!
  ...

    I.e.:, the PC contains the return address !

Illustrating the effect of the bl (branch and link) instruction

        mov     r0, #1111
        mov     r1, #1111
        bl      myMethod                         
        mov     r2, #2222
        mov     r3, #2222


myMethod:
        mov     r8, #4444
        mov     r9, #4444

Illustrating the effect of the bl (branch and link) instruction 

main:
        bl      myMethod

        mov     r8, #4444

When the CPU executes bl myMethod, look for:

  1. PC = address of myMethod

    I.e.: the program made a jump to the myMethod function

    (So the instructions in the myMethod function will now be executed)

  2. LR register contains: the address of the mov r2, #2222 instruction
