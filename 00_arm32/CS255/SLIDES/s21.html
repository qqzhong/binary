Statements in a high level programming language

There are 3 kinds of statements in any high level programming language:

  1. Assignment statements

  2. Conditional statements (if, if-else, ...)

  3. Loop statements (while, for, ....)

 

You can prove that programming languages that contain these 3 statement types are Turing complete.

 

We have learned enough assembler programming to show you how the Java compiler translates programs that contains assignment statements using simple (= unstructured) variables

Jave program with simple (class) variables and an assignment statement

The complete ARM assembler program: 

     .text
main:
  movw    r0, #:lower16:a  // Moves the address of memory
  movt    r0, #:upper16:a  // variable a into register r0
  ldr     r1,[r0]          // Load int value from a into r1

  // Move b into r2
  movw    r0, #:lower16:b  // Moves the address of memory
  movt    r0, #:upper16:b  // variable b into register r0
  ldr     r2, [r0]         // Load int value from b into r2
  mov     r3, #5
  mul     r2, r3, r2       // r2 = 5*b

  // Add them up
  add     r2, r1, r2       // r2 = a + 5*b

  // Move sum in r2 to c 
  movw    r0, #:lower16:c  // Moves the address of memory
  movt    r0, #:upper16:c  // variable c into register r0
  str     r2,[r0]          // Store int value r2 to c 

  .data
 a:  .4byte  4    // a initialized with the value 4
     .align  2
 b:  .4byte  5    // b initialized with the value 5
     .align  2
 c:  .skip   4    // c is not initialized  
 .end

Mixed data type operations

Compiler records the type information on all variables in the program: 

   public class myProg {
      static byte   a = 4;        
      static short  b = 5;        
      static int    c;            

      public static void main( )
      {
         c = a + 5*b;                   
      }
   }

Will cause the compiler to use different variable definition directives and load (and store) instructions
