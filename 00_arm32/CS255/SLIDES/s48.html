Let's try to implement (= realize) function call + return using known assembler instructions
 

Function call + return is the program flow of a function call + return from the called function

The function call + return program flow is as follows (CS170 material): 

   function f1( )               function f2( )
   {                   +------->{         
       ...             |            ...  |
       ...             |            ...  |
       ...       call  |            ...  |
       f2( ) ----------+            ...  |
       ...   <--------------+       ...  |
       ...                  |       ...  V 
       ...                  +------ return;
   }                             }

Why the "branch always" instruction cannot be used to implement function call + return

Example where you cannot use the b instruction to implement the function call + return flow: 

   main( )
   {
      ...
      A( );          
      ...   <---- return location of fist call
      ...
      A( );   
      ...   <---- return location of second call
   }

   A( )
   {
      ...
      ...  <-- Using "b label" (label is a fixed location)
               A( ) can only return to one specific location ! 
   }

Requirement: the subroutine call instruction must record the return location !!!

Lessons learned from these function call + return examples
 

  • Calling a subroutine:

      • The call-subroutine instruction (that jumps to the start of a subroutine) must also save the return location (= address) (which is the location right after the subroutine call instruction) 
            call   subroutineName
    next-instruction    <--- return location

    This return location is called: return address

  • How a subroutine returns to its caller:

      • The subroutine-return instruction must jump to the return address saved by the (lastest) call-subroutine instruction

Analogy of the function call + return program flow
 

  • The call-subroutin instruction must "leave a bread crump" to indicate where to return

  • The subroutine-return instruction must "consume the (lastest) bread crump" to jump to the (lastest) return location
